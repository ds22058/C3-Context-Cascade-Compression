## 一、单选题（7 题）

### 1. 数学类 | 简单 | 单选
设 `σ(x) = 1 / (1 + e⁻ˣ)`，则 `σ′(x)` 等于：

A. `σ(x)`  
B. `1 - σ(x)`  
C. `σ(x) · (1 - σ(x))`  
D. `e⁻ˣ / (1 - e⁻ˣ)`

答案：C

解析：  
sigmoid 的经典导数形式为：

```text
σ(x) = 1 / (1 + e⁻ˣ)
σ′(x) = σ(x) · (1 - σ(x))
```

这是神经网络里非常常见的基础结果。

### 2. Transformer | 简单 | 单选
在标准 Transformer 中，位置编码加入 token embedding 的最直接目的是什么？

A. 让不同 batch 的长度一致  
B. 给模型提供相对或绝对的位置信息  
C. 降低 softmax 的计算量  
D. 提高词表覆盖率

答案：B

解析：  
自注意力本身不区分顺序，因此需要额外注入位置信息。无论是绝对位置编码还是相对位置偏置，本质上都是让模型“知道”顺序。

### 3. Transformer | 简单 | 单选
如果某层自注意力的输入张量形状为 `n × d_model`，那么每个 attention head 在投影后常见的 Query 维度通常是：

A. `n`  
B. `d_model`  
C. `d_model / h`  
D. `n / h`

答案：C

解析：  
如果一层有 `h` 个头，那么常见设置是每个 head 的维度 `d_k = d_model / h`。这样多个 head 拼接后仍能回到原始表示维度。

### 4. 数学类 | 中等 | 单选
设一元函数 `f(x)` 二阶可导，且对任意 `x` 都有 `f″(x) ≥ 0`，则下面结论通常成立的是：

A. `f(x)` 一定单调递增  
B. `f(x)` 一定有唯一极小值  
C. `f(x)` 是凸函数  
D. `f(x)` 是奇函数

答案：C

解析：  
在一元二阶可导条件下，`f″(x) ≥ 0` 是函数凸性的充分条件。因此 C 正确。  
A 不一定，因为导数可以先负后正。  
B 不一定，凸函数不一定存在唯一极小值。  
D 与二阶导非负无关。

### 5. Transformer | 中等 | 单选
在 scaled dot-product attention 中，如果不做 `1 / √d_k` 缩放，最可能带来的问题是：

A. Q、K、V 无法学习  
B. attention 输出维度发生变化  
C. softmax 更容易饱和，训练稳定性变差  
D. 多头注意力失效为单头注意力

答案：C

解析：  
当 `d_k` 增大时，点积的方差会增大，进入 softmax 后更容易出现“非常尖锐”的分布，导致梯度偏小，训练不稳定。因此需要除以 `√d_k`。

### 6. Transformer | 中等 | 单选
为什么 Transformer 的前馈网络 FFN 往往对每个位置独立应用，但仍然能提升序列表达能力？

A. 因为 FFN 会显式交换不同位置的信息  
B. 因为位置间的信息已在注意力子层中完成交互，FFN 负责逐位置非线性变换  
C. 因为 FFN 只在 Decoder 中生效  
D. 因为 FFN 等价于卷积核大小为序列长度的卷积

答案：B

解析：  
位置之间的交互主要由 attention 完成，而 FFN 更像对每个位置做共享参数的非线性特征变换。两者结合后，既能建模位置间关系，也能增强单位置表征能力。

### 7. Transformer | 中等 | 单选
在自回归解码时，若不使用 KV Cache，而是在每一步都重新计算历史 token 的注意力投影，则推理过程相较使用 KV Cache 通常会：

A. 更快，因为避免了缓存访存  
B. 更慢，因为重复计算历史 token 的 Key 和 Value  
C. 完全等价，没有明显差别  
D. 更稳定，因为数值误差更小

答案：B

解析：  
KV Cache 的本质是在历史 token 不变时复用其 Key 和 Value。若每一步都重新算一遍历史部分，就会引入大量重复计算，因此推理会明显变慢。

## 二、多选题（3 题）

### 8. 数学类 | 中等 | 多选
关于 softmax 函数 `softmax(z)` 的性质，下列说法正确的是：

A. 对任意常数 `c`，`softmax(z)` 与 `softmax(z + c·1)` 相同  
B. softmax 输出各分量都严格大于 0  
C. softmax 输出各分量之和等于 1  
D. 对所有输入向量，softmax 一定产生 one-hot 输出

答案：A、B、C

解析：  
softmax 的定义为：

```text
softmax(zᵢ) = eᶻⁱ / Σⱼ eᶻʲ
```

- 同时给所有分量加上同一个常数 `c`，分子分母会同时乘上 `eᶜ`，结果不变，所以 A 正确。
- 指数函数始终大于 0，因此每个输出分量都大于 0，所以 B 正确。
- softmax 输出本质上是一个概率分布，因此总和为 1，所以 C 正确。
- softmax 只是在极端情况下接近 one-hot，一般并不严格等于 one-hot，所以 D 错误。

### 9. Transformer | 困难 | 多选
关于 Pre-Norm Transformer 与 Post-Norm Transformer，下列说法正确的是：

A. Pre-Norm 一般更利于深层模型训练稳定性  
B. Post-Norm 中，LayerNorm 位于残差相加之前  
C. Pre-Norm 中，子层输入先做归一化，再进入 attention 或 FFN  
D. 两者在浅层网络中一定表现完全相同

答案：A、C

解析：  
- Pre-Norm 常见形式是先归一化、再进入子层、最后做残差相加，通常更利于深层训练稳定，所以 A 正确，C 正确。
- Post-Norm 是子层输出与残差相加后再做归一化，因此 B 错误。
- 二者并不保证在浅层网络中“完全相同”，只是有时性能接近，因此 D 错误。

### 10. Transformer | 困难 | 多选
关于注意力复杂度与工程实现，下列说法正确的是：

A. 标准全连接自注意力对序列长度 `n` 的时间复杂度通常可视为 `O(n²)`  
B. 标准全连接自注意力的注意力分数矩阵大小通常与 `n²` 同阶  
C. 使用 causal mask 后，Decoder 自注意力的理论复杂度会从 `O(n²)` 严格降为 `O(n)`  
D. 在长序列场景下，显存压力往往不仅来自参数，还来自中间激活和 attention 矩阵

答案：A、B、D

解析：  
- 对长度为 `n` 的序列，全连接自注意力需要计算大量两两交互，因此时间复杂度通常视为 `O(n²)`，A 正确。
- attention score 矩阵通常是 `n × n`，所以空间规模同样与 `n²` 同阶，B 正确。
- causal mask 只是把未来位置屏蔽掉，并没有从根本上把理论复杂度严格改成 `O(n)`，所以 C 错误。
- 实际工程中，长序列训练或推理时，显存瓶颈经常来自 activation、KV cache 或 attention matrix，而不只是模型参数本身，所以 D 正确。
