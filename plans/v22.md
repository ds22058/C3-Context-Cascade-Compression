# C3 v2.2 训练方案

## 1. 训练数据

### 1.1 数据来源

| FreeLaw | `pile-of-law/pile-of-law` (courtlistener) | 法律文书 | Public Domain |

### 1.2 数据池与配比

采用 **三个数据池**：

- **重建池**：文本 >= 7168 tokens → 随机截断到 [7168, 9216]
- **续写池**：文本 >= 16384 tokens → 截断到 16384, 在 [7168, 9216] 处 split
- **指令池**：长上下文指令 (context >= 7168 tokens) + 拼接短指令 → Phase 2 独占

特殊处理：
- PG-19：书籍分 16k chunk
- UltraChat / OpenHermes：多条对话拼接到 7168+
- LongAlign / LongAlpaca：天然长上下文，直接使用

### 1.3 上下文长度

| 任务 | 原文要求 | 截断范围 | 处理方式 |
|------|---------|---------|---------|
| 重建 | >= 7168 tokens | **[7168, 9216]** 随机 | 随机截断到该范围，avg ~8192 |
| 续写 | >= 16384 tokens | 截断到 **16384** | 在 **[7168, 9216]** 处随机 split |
| 指令 | context >= 7168 | **[7168, 9216]** 随机 | context 截断到该范围, response 保持原长 |

续写 / 指令 split 后：
- 上半部分 (context)：7168-9216 tokens → 送入 encoder 压缩为 512 latent tokens
- 下半部分 (generation)：续写 7168-9216 tokens / 指令 response 变长 → decoder 预测目标

### 1.4 两阶段数据配比

| 阶段 | 重建 | 续写 | 指令 | 总量 |
|------|------|------|------|------|
| **Phase 1** | ~70 万 (100%) | 0 | 0 | ~70 万 |
| **Phase 2** | ~50 万 (60%) | ~24 万 (29%) | ~4 万 (5%) | ~78 万 |

Phase 2 指令数据仅占 5%，作为辅助任务不会稀释核心压缩能力，同时赋予模型指令遵从能力。

## 2. 训练设置

### 2.1 模型架构

```
Encoder (Qwen2.5-1.5B, C3 预训练)
  → Q Queries (512 个 latent tokens)
  → mm_projector (1536 → 2048)
  → Decoder (Qwen2.5-3B, 原始权重)
```

- **压缩比**：~8192 tokens → 512 latent tokens = **16:1 压缩比**
- `latent_token_len` 通过 `--latent_token_len` 参数配置（默认 512）
- 位置编码上限：32768（Qwen2.5 默认），远超训练序列长度

### 2.2 任务描述

#### 重建任务 (reconstruct)
- **Encoder 输入**：原文 + `<img><imgpad>×512</img>` 占位符
- **Decoder 输入**：`[system] [user: <placeholder> + "Repeat the text: "] [assistant: 原文]`
- **训练目标**：decoder 从 512 个 latent tokens 重建完整原文
- **Labels**：仅对 assistant 部分计算 loss

#### 续写任务 (continuation)
- **Encoder 输入**：前半段文本 (7168-9216 tokens) + latent 占位符
- **Decoder 输入**：`[system] [user: <placeholder>] [assistant: 后半段文本]`
- **训练目标**：decoder 从 512 个压缩 latent tokens 续写后半段 (7168-9216 tokens)
- **Labels**：仅对 assistant 部分计算 loss

#### 指令任务 (instruction) — Phase 2 only
- **不走压缩**：指令上下文直接放在 decoder user message 中，不经过 encoder 压缩
- **Decoder 输入**：`[system] [user: 完整上下文文本] [assistant: 回复]`
- **训练目标**：decoder 从完整上下文直接生成回复（标准 causal LM）
- **Labels**：仅对 assistant 部分计算 loss
- **Encoder**：context_ids 仍经过 encoder（保持 batch 兼容），但 latent 结果不注入 decoder（input_ids 无 `<img>` 占位符）
- 数据格式复用 continuation 的 split_ratio（context + response 拼接）

### 2.3 训练超参数

| 参数 | Phase 1 | Phase 2 |
|------|---------|---------|
| **可训参数** | encoder + Q + mm_projector (~1.5B) | 全部 (~4.5B) |
| **冻结** | decoder (Qwen2.5-3B) | 无 |
| **数据** | ~70 万 reconstruct | ~50 万 recon + ~24 万 cont + ~4 万 instruct |
| **Epochs** | 5 (可选 3) | 3 |
| **Learning Rate** | 1e-5, cosine, warmup 100 steps | 1e-5 (encoder 和 decoder 均为 1e-5) |
| **Effective Batch** | 4 × 8 × 8 = 256 | 4 × 8 × 8 = 256 |
| **MAX_LENGTH** | 12288 | 12288 |
| **latent_token_len** | 512 | 512 |
| **精度** | bf16 | bf16 |
| **Gradient Checkpointing** | True | True |
| **group_by_length** | True | True |
| **FlashAttention** | flash_attention_2 | flash_attention_2 |
| **DeepSpeed** | ZeRO-2 | ZeRO-2 |

### 2.4 与 v2.0 对比

| 维度 | v2.0 | v2.2 |
|------|------|------|
| 重建上下文 | 400-1500 tokens | **7168-9216 tokens** |
| 续写总长 | 800-2000 tokens | **16384 tokens** |
| latent_token_len | 32 | **512** |
| 压缩比 | ~1500/32 ≈ 47:1 | **~8192/512 = 16:1** |
| 数据源 | 4 源 | **13 源** |
| 续写 split | ratio 0.4-0.7 | **position [7168, 9216]** |
| 指令数据 | 无 | **~4 万 (Phase 2, 5%)** |
| Recon tokens/epoch | 20 亿 | **57 亿 (2.9x)** |
| Cont tokens/epoch | 9 亿 | **49 亿 (5.4x)** |

### 2.5 文件路径

```
data/
  download_data_v22.py            ← 10 源主下载
  download_supplement_v22.py      ← 5 源补充下载 (arxiv + FreeLaw增量 + 指令数据)
  prepare_data_v22.py             ← token 级截断 + split + 指令处理
  run_data_pipeline_v22.sh        ← 一键数据管线
  raw_v22/                        ← 原始数据
    *_recon.jsonl, *_cont.jsonl   ← 重建/续写池
    *_instruct.jsonl              ← 指令池 (context + response 分离)
  processed_v22/
    phase1_train.jsonl            ← ~70 万 reconstruct
    phase2_train.jsonl            ← ~50 万 recon + ~24 万 cont + ~4 万 instruct
    continuation_pool.jsonl       ← 完整续写池 (备用)
    instruction_pool.jsonl        ← 完整指令池 (备用)

train/
  run_phase1_v22.sh               ← Phase 1 单机脚本
  run_phase2_v22.sh               ← Phase 2 单机脚本
  run_phase2_multinode.sh         ← Phase 2 多机脚本
```
